我的自用 软件 和 服务 清单

这是一个 持续更新 的清单，记录我在 各个平台（目前主要是 Windows 、 Linux 和 Android） 的首选 软件 / App / 服务。供我换机或重装系统时 回顾，也供大家参考；也欢迎大家在留言推荐更好的选择。

选择标准：

  • 好用：这是一个超级主观的标准，只能说适合我，我会谈谈到底哪里好，但不一定适合其他人

  • 开源：省钱固然是一方面[:-D]，但只是一部分原因。更重要的，第一是 『林纳斯定律』 ，一个社区活跃的开源软件，会有更少的bug，快速的演化，以及几乎不可能有后门。（当然也并不绝对，参见『Ken Thompson 大神的C语言编译器后门』，最好工具引用的技术也都是开源的）第二是开源的特性决定了，即使原作者 / 版权方 放弃更新，或者转向闭源，足够活跃的社区依然可以维持开源分支的开发（例如 Hudson > Jenkins， MySQL > MariaDB），开源的软件能使用更长时间，不会因为商业上的原因突然中断。一个开源工具沉寂之时，往往是被另一个更优秀的工具代替了。

    在此基础上，拒绝所有破解版工具。如果一个工具实在没有开源 / 免费 的替代品，那就购买。使用破解软件，一方面存在安全风险，另一方面有违软件开发者的立场。

    基于类似原因，推荐大家都到官网下载，谨慎使用所谓的绿化软件，如果实在有这种需求可以考虑自己用UniversalExtractor制作。

  • 跨平台一致性：需要在不同的平台上使用软件，已经是一个不可回避的现实。所以如果有两个软件各方面都差不多，我会优先选择那个跨平台，而且 在不同平台上使用体验一致 的。因为这样会降低学习和来回切换平台的成本。

我一开始是先列平台，然后在三个平台下分别列需求，然后列软件。后来发现，由于某些优秀跨平台软件的存在，其实几个平台都用一个……这样就会很多重复内容，所以改为先列需求，再在需求下列软件。曾经用过的软件如果比较优秀也会列出，当前已经不用或者很少用的，会加上删除线。只有标题的，先占位，慢慢补充。

注意这是一个清单，只列出简介，不涉及用法，另有一篇 小工具专题 分享小工具的使用。

文本编辑器

因为我主要使用Markdown作为我的文档原稿，所以Markdown编辑器单独列出。（参见 为什么你应该用 Markdown 写文档？ ， 其他标记语言也偶尔使用，但对于绝大多数场景来说，都不如Markdown轻巧，也不如Markdown的工具支持丰富。日后如果Markdown的方言陷入兼容性问题，考虑往AsciiDoc 迁移。）

Markdown

目前主力用Typora，Smark备用。

  • Typora (Windows / Linux / Mac) ： 非常干净Markdown编辑器，当前首选。

    • 优点 ： 功能强大，基本上该有的都有了，导出选项也很丰富（其实后端是pandoc）。同时界面非常简洁美观，支持多种写作模式，尽量减少对你的打扰，让你专注在写字这件事上。甚至让我觉得用它写东西已经是一种享受。如果不是它出现，我本来觉得Smark已经很好了。这很难描述，需要自己体验才知道。
    • 缺点 ： 虽然还在beta阶段，但没发现什么问题，感觉挺稳定，更新也很勤快，刚支持完Linux，马上又推出了 Mac的支持。非要说的话，不开源，而且稳定版之后存在收费的可能（？）。

  • Smark(Github) / Smark(OSChina)(Windwos / Linux / Mac) ：作者为自己写的pandoc前端。

    • 优点 ： 以pandoc为后端，基本上pandoc的优点它都继承了，主要是格式支持丰富。但是pandoc动不动要命令行操作被UI所取代。而且由于它基于的pandoc和Qt都是跨平台的，所以它可以保持多平台的体验一致性。我对它已经非常满意了，要不是后来发现了Typora，我可能就一直用着它了。
    • 缺点 ： 不是实时刷新。不过这点其实问题不大，先专心写一段，想看预览点一下F5也不是什么大问题。另外一个就是，虽然跨平台，作者只编译打包了Win32的包（给他自己用）。Win64需要自己替换64位的pandoc，其他平台需要自己配置Python + PyQt 环境之后编译。我目前在Ubuntu上编译了一个，其他平台还没试过。

  • pandoc (跨平台)：pandoc 号称 标记语言的瑞士军刀，可以在多种标记语言和文档格式之间自由切换，Markdown只是它支持的其中一种语言。不过是一个命令行工具，一般用户使用会有不便。上面两个优秀的编辑器都有使用pandoc作为后端。

  • MarkdownPad (Windows, 需要.NET)：早期Windows上选择不多的时候的选择。
    • 优点：简单易用，实时刷新，复制可以选择复制Markdown还是HTML，而且HTML会自动带上CSS。有一段在公司不支持Markdown的论坛上面写东西的时候，直接HTML连CSS复制很省心。
    • 缺点：功能很少，导出选项很少。而且就这样，还动不动就要收费版才能用。现在用来后备，基本不用了。

文本编辑

大文件加载快，支持多编码多格式（例如自动识别Windows和Linux的换行），能写字，也能在写段小脚本懒得开IDE时编辑代码，或小巧或功能强大。

目前Linux环境用 Vim，Windows 环境主用 Notepad++，但是默认打开方式设为Notepad2（因为更轻巧）。后面想试用一下Atom，再回来更新。

更多编辑器比较，可以参考 维基百科:文件编辑器比较

  • Vim (跨平台)：老牌、强大、经典，只要你愿意，通过增加插件和配置，好像没有做不到的事情，Linux下的首选编辑器，我习惯用它纯粹只是习惯，无意与Emacs党争论。

  • Notepad++ (Windows) ：Windows 下支持各种语法高亮、宏和插件的编辑器多面手。默认就很强大，也可以轻易通过配置和插件变得更适合自己。个人觉得多行编辑是比较少见的特色。

  • Notepad2 (Windows) ： 百K单文件的轻量级软件，系统 Notepad 的最佳替代品。

  • SciTE(Scintilla , 跨平台) ： SciTE 是 SCIntilla based Text Editor 的缩写，一开始只是作者作为Scintilla 使用范例写的。 Scintilla 是一个用C++ 写的开源编辑器组件，因为太过优秀，一经出现，之后就很少有人会重头写编辑器了，都是基于Scintilla 加一个外壳。这里有一个很长的 列表 列出基于 Scintilla 的编辑器，里面就包括了 Notepad++ 和 Notepad2。作为 Scintilla 嫡亲的 SciTE 当然也很优秀，单文件就一经非常强大，也有极强的可配置性，还能通过Lua脚本扩展功能。

    引用 小众软件 的话： 『较其它同重量级软件，最耀眼的就是导出功能，可以导出 PDF/HTML/RTF/XML/LaTex 类型的文件，直接就能将语法高亮的内容导出』。但对于中文用户有个比较致命的缺点，就是中文支持不稳定。所以放在这里致敬，但并不是十分推荐。

  • Sublime Text(跨平台) ： 个人感觉各方面都很像Notepad++，不过颜值更高一些。本身是专有软件，但是上面很多扩展包都是开源的，因为社区的活跃而变得强大。严格来说，使用Sublime Text 需要购买授权，但是比较良心的提供不限期试用，导致很多人一直不注册。授权方式比较灵活，可以按人数，也可以按机器，官方甚至贴心地推荐按钱少的方式买；个人授权可以用于工作环境，但反过来不行。如果你觉得它好用，最好付费使用，不然就选择开源免费的工具。
